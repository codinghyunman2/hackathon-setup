# 입력 템플릿 (한국어)

> 사용자에게 Step 1에서 그대로 보여주는 6섹션 템플릿입니다. 각 섹션 아래에 **예시 답변 1줄**이 회색 글씨로 들어가 있어 빈칸 공포를 줄입니다.

---

## 이렇게 답하시면 돼요!

- 아래 6개 섹션을 보고 **불렛포인트로 자유롭게** 한국어로 답해주세요.
- 어려운 항목은 **비워두거나 "잘 모르겠어요"** 라고 써도 괜찮아요. 클로드가 다음 단계에서 물어볼게요.
- **필수**: 문제 / 원하는 것 / 사용 범위 — 이 3개는 꼭 한 줄이라도 채워주세요.
- **선택**: 나에 대해 / 데이터 / **만들 방식** / 기타 — 답하면 더 좋지만 없어도 진행 가능해요. 만들 방식은 잘 모르면 "추천받을게요"라고 쓰면 클로드가 PRD 초안 본 뒤에 추천해드려요.

---

# [필수]

## 🔴 문제
- 지금 불편한 상황:
  - 예) "광고 채널별 성과 리포트를 매주 수작업으로 만들고 있어요"
- 얼마나 자주 발생하는지:
  - 예) "매주 월요일 오전, 약 3시간 소요"
- **지금 손으로 하는 단계 3~5개** (★ 자동화 어디를 맡길지 정하는 핵심):
  - 예)
    1. Meta Ads 들어가서 지난주 캠페인별 CSV 다운로드
    2. 구글시트 템플릿에 붙여넣고 함수로 ROAS 계산
    3. 슬랙 #marketing-weekly 채널에 결과 메시지 복붙
  - "단계가 잘 안 떠올라요" 라고 써도 OK — 클로드가 짐작해서 초안 보여드려요

---

## 🟢 원하는 것
- 자동화/해결하고 싶은 것:
  - 예) "매주 자동으로 리포트 생성되게 하고 싶어요"
- 결과물 형태 (알림? 리포트? 대시보드? 파일?):
  - 예) "구글 슬라이드 형식의 주간 리포트"
- 결과물이 나오는 곳 (슬랙? 이메일? 구글 시트? 웹?):
  - 예) "팀 슬랙 채널 #weekly-report 에 자동 공유"

---

## 👥 사용 범위
- 사용자: 혼자 / 팀 내부 / 외부 공개
  - 예) "팀 내부 (마케팅팀 8명)"
- 사용 인원 (명):
  - 예) "8명"
- 배포 방식: 로컬 실행 / 사내 서버 / 외부 배포 (Vercel 등)
  - 예) "로컬 실행 (내 노트북에서 매주 실행)"

---

# [선택]

## 📝 나에 대해
- 직무/역할:
  - 예) "성과 마케터, 디지털 광고 운영"
- 소속 팀/회사 규모:
  - 예) "마케팅팀 8명 / 총 80명 규모 스타트업"

---

## 📊 데이터
- 데이터 출처 (어디서 가져오는지):
  - 예) "Meta Ads, Google Ads, Naver Search Ads"
- 실행 주기 (매일? 매주? 수동 실행?):
  - 예) "매주 월요일 오전 9시 자동 실행"
- 연동된 툴/시스템 (Meta, Airbridge, Notion 등):
  - 예) "Airbridge, Google Sheets, Slack"

---

## 🛠️ 만들 방식
> 💡 **잘 모르겠으면 비워두거나 "추천받을게요" 라고 쓰세요.** PRD 초안이 나오면 클로드가 두 가지 중 어떤 게 더 적합한지 1줄 추천해드려요.
- 어떤 방식으로 만들고 싶어요?
  - `Claude Code (코드 작성)` — 클로드와 대화하면서 코드 작성. 커스텀 로직/AI 호출/복잡한 데이터 가공에 강해요.
  - `n8n (노드 연결, 노코드)` — n8n 인스턴스에서 노드 드래그앤드롭. 외부 API 여러 개 연동에 강해요.
  - `추천받을게요` ← **잘 모르면 이걸 고르세요. Step 3에서 클로드가 추천 후 최종 결정.**
- (참고) n8n 인스턴스를 회사에서 운영 중이거나 본인이 쓸 수 있는 환경인가요? "네 / 아니요 / 모르겠어요" 한 줄이면 추천 정확도가 올라가요.
