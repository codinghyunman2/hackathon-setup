# 입력 템플릿 (한국어)

> 사용자에게 Step 1에서 그대로 보여주는 6섹션 템플릿입니다. 각 섹션 아래에 **예시 답변 1줄**이 회색 글씨로 들어가 있어 빈칸 공포를 줄입니다.

---

## 이렇게 답하시면 돼요!

- 아래 6개 섹션을 보고 **불렛포인트로 자유롭게** 한국어로 답해주세요.
- 어려운 항목은 **비워두거나 "잘 모르겠어요"** 라고 써도 괜찮아요. 클로드가 다음 단계에서 물어볼게요.
- **필수**: 문제 / 원하는 것 / 사용 범위 — 이 3개는 꼭 한 줄이라도 채워주세요.
- **선택**: 나에 대해 / 데이터 / 기타 — 답하면 더 좋지만 없어도 진행 가능해요.

---

## 나에 대해 (선택)
- 직무/역할:
  - 예) "성과 마케터, 디지털 광고 운영"
- 소속 팀/회사 규모:
  - 예) "마케팅팀 8명 / 총 80명 규모 스타트업"

## 문제 (필수)
- 지금 불편한 상황:
  - 예) "광고 채널별 성과 리포트를 매주 수작업으로 만들고 있어요"
- 얼마나 자주 발생하는지:
  - 예) "매주 월요일 오전, 약 3시간 소요"
- 현재 어떻게 해결하고 있는지 (수동? 엑셀? 등):
  - 예) "각 광고 플랫폼에서 CSV 받아 구글 시트에 복붙 + 함수로 가공"

## 원하는 것 (필수)
- 자동화/해결하고 싶은 것:
  - 예) "매주 자동으로 리포트 생성되게 하고 싶어요"
- 결과물 형태 (알림? 리포트? 대시보드? 파일?):
  - 예) "구글 슬라이드 형식의 주간 리포트"
- 결과물이 나오는 곳 (슬랙? 이메일? 구글 시트? 웹?):
  - 예) "팀 슬랙 채널 #weekly-report 에 자동 공유"

## 데이터 (선택)
- 데이터 출처 (어디서 가져오는지):
  - 예) "Meta Ads, Google Ads, Naver Search Ads"
- 실행 주기 (매일? 매주? 수동 실행?):
  - 예) "매주 월요일 오전 9시 자동 실행"
- 연동된 툴/시스템 (Meta, Airbridge, Notion 등):
  - 예) "Airbridge, Google Sheets, Slack"

## 사용 범위 (필수)
- 사용자: 혼자 / 팀 내부 / 외부 공개
  - 예) "팀 내부 (마케팅팀 8명)"
- 사용 인원 (명):
  - 예) "8명"
- 배포 방식: 로컬 실행 / 사내 서버 / 외부 배포 (Vercel 등)
  - 예) "로컬 실행 (내 노트북에서 매주 실행)"

## 기타 (선택)
- 비슷한 레퍼런스 (있으면):
  - 예) "Looker Studio 주간 리포트 같은 느낌"
- 절대 안 됐으면 하는 것 (있으면):
  - 예) "광고비 원본 데이터를 외부 서버에 올리는 건 안 돼요"
